## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability, please follow these steps:

### 1. **DO NOT** create a public GitHub issue

Security vulnerabilities should be reported privately to prevent exploitation.

### 2. Contact Information

Please report security vulnerabilities via one of the following channels:

- **Email**: [Add security email if available]
- **Telegram**: [@Tech Genie](https://t.me/opensea712)
- **Twitter**: [@Tech Genie](https://x.com/shinytechapes)

### 3. Include the Following Information

When reporting, please include:

- **Description**: Clear description of the vulnerability
- **Impact**: Potential impact and severity
- **Steps to Reproduce**: Detailed steps to reproduce the issue
- **Proof of Concept**: If possible, include a proof of concept
- **Suggested Fix**: If you have ideas for a fix, please share them

### 4. Response Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: Depends on severity and complexity

### 5. Severity Levels

We use the following severity classification:

- **Critical**: Immediate threat to funds or protocol integrity
- **High**: Significant risk requiring urgent attention
- **Medium**: Important issue that should be addressed
- **Low**: Minor issue with limited impact

## Security Best Practices

### For Users

- Always verify contract addresses before interacting
- Use official frontends and verified contracts
- Never share private keys or seed phrases
- Be cautious with approvals and permissions
- Review transaction details before signing

### For Developers

- Never commit private keys or secrets
- Use environment variables for sensitive data
- Review all dependencies for vulnerabilities
- Follow secure coding practices
- Conduct thorough testing before deployment

## Responsible Disclosure

We follow responsible disclosure practices:

1. **Private Reporting**: Report vulnerabilities privately first
2. **Confirmation**: We'll confirm receipt and assess the issue
3. **Fix Development**: We'll develop and test a fix
4. **Coordination**: We'll coordinate disclosure timing
5. **Public Disclosure**: After fix is deployed, we'll publicly acknowledge (with your permission)
